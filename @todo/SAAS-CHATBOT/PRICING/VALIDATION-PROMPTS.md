# PRICING — Per-file validation prompts

> Prompts (in English — consumed by an LLM reviewer, so English saves tokens) to **re-validate each
> volatile file in PRICING/ independently**. The whole point of the split is per-file re-validation:
> run one prompt to re-audit one file. Mirrors the tone of the root `../VALIDATION-PROMPTS.md`.
> Last updated: 2026-06-14
>
> **Shared shape of every prompt:** role + context to read + scope + how to respond (severity
> findings 🔴/🟡/🟢, show the math, diff old→new, separate FACT from ASSUMPTION).
>
> **Tooling note:** the live price data is regenerated by `extract/fetch-openrouter-pricing.sh` and
> inspected via `extract/openrouter-pricing.html`. Several prompts below ask you to run it.

---

## 🏗️ PROMPT A — `infrastructure.md` (provider tiers + scale stages)

```
You are a cloud cost analyst re-validating the infra cost model of a whitelabel RAG chatbot SaaS.

CONTEXT TO READ:
1. @todo/SAAS-CHATBOT/PRICING/infrastructure.md — §1.1 provider tiers + §1.2 infra by scale stage

SCOPE:
- Re-validate each provider's tiers LIVE (pricing pages): Supabase, Railway, Vercel, Cloudflare
  (Pages/R2), Upstash QStash, Stripe (card/PIX/boleto fees).
- Did any tier name, price, included quota, or free-tier policy change? (e.g. Railway free tier,
  Supabase idle-pause, R2 egress staying free, Stripe PIX 1.19%).
- Does the §1.2 scale-stage mapping (MVP/Early/Growth/Scale) still pick the right tier per load, and
  do the `~Total/mo` per stage still add up from the §1.1 tables?

HOW TO RESPOND:
1. Executive summary (3–5 lines): is the infra model still accurate? Biggest drift?
2. "old → new" diff per service (price/tier/quota), with the source URL.
3. Recompute each stage `~Total/mo`; flag any that moved and the downstream impact on plans.md margins.
4. Findings by severity 🔴/🟡/🟢 (file/section, impact, fix).
Rules: show the math, separate FACT from ASSUMPTION, cite the pricing page.
```

## 🤖 PROMPT B — `models.md` (generation mix + routing spread)

```

whitelabel RAG chatbot SaaS.

CONTEXT TO READ:
1. @todo/SAAS-CHATBOT/PRICING/models.md            — §1.5 model layer + §8 router (§8.1, §8.2)

- Open `@todo/SAAS-CHATBOT/PRICING/extract/openrouter-pricing.html` and review the **Our models**,
  **Newest** and **All models** tabs (note NEW/PROMO badges and price buckets).

SCOPE:
- Re-validate live prices for the anchor (Sonnet 4.6) + the 80/15/5 mix (Qwen3.7 Plus / DeepSeek V4
  Pro / Sonnet 4.6). Did any price change? Are promos still active or did they expire?

- **Newest → benchmark hook:** list any model the "Newest" tab surfaces that the routing mix should
  consider, AND any model in models.md marked "n/d" (e.g. Qwen3.7 Plus/Max) that still lacks a score

HOW TO RESPOND:
1. Executive summary (3–5 lines): does the ~85% spread still hold? Biggest risk?
2. "old → new" price diff per model (with promo flag).
3. Recomputed blended cost + spread + the full §8.2 table (corrected numbers if they diverge).
4. List of new/un-scored models to benchmark (feed to model-benchmark.md).
5. Findings by severity 🔴/🟡/🟢.
Rules: ALWAYS show the math, re-validate live, separate FACT from ASSUMPTION.
```
